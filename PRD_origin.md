## 1. 개요

### 1.1 배경

- 산업안전보건법 제36조는 모든 사업주에게 유해·위험요인을 파악하고 위험성을 평가·개선할 의무를 부과하며, 결과와 조치사항의 **3년 기록·보존**(시행규칙 제37조)을 요구한다.
- 2023년 고시 개정(제2023-19호)으로 **상시평가 제도**(월·주·일 주기)가 신설되었고, 2024년 개정(제2024-76호)으로 참여·공유 대상이 **고용형태·국적 불문 사업장 내 모든 근로자**(협력사 포함)로 명확화되었다.
- 중대재해처벌법 시행령 제4조 제3호에 따라 위험성평가 절차를 마련하여 실시하고 그 결과를 경영책임자가 보고받으면 반기 1회 유해·위험요인 확인·개선 점검 의무를 이행한 것으로 간주된다. → 위험성평가 기록은 중처법 대응의 **핵심 증빙**이다.
- 현재 엑셀·수기 중심의 평가 관리로는 복수 사업장·다수 협력사의 평가 이력, 결재, 개선조치 이행을 일관되게 추적하기 어렵고, 법정 기한(정기평가 연 1회, 수시평가 트리거) 누락 위험이 있다.

### 1.2 제품 목적

건설업·제조업 복수 사업장을 운영하는 기업이 **자사 근로자와 협력사(수급업체) 근로자가 함께 사용하는 개방형 시스템**에서:

1. HAZOP·KRAS·JSA·CHARM 4개 기법의 위험성평가를 작성 → 결재 → 공유 → 기록까지 온라인으로 일원화하고,
2. 감소대책(개선조치)의 이행을 담당자·기한·증빙·재평가까지 추적하며,
3. 최초·정기·수시·상시평가의 법정 실시 주기를 자동 관리하고,
4. 본사가 전 사업장·협력사의 이행 현황을 통합 대시보드로 관제한다.

### 1.3 제품 구조 원칙 (승인된 설계 방향)

**공통 평가 엔진 + 기법별 템플릿** 구조를 채택한다.

- 결재, 이행관리, 공유·서명, 기록·보존, 알림, 일정 관리는 4개 기법이 **하나의 공통 워크플로**를 공유한다.
- 기법별로 다른 것은 **평가 시트(입력 폼)의 필드 구성과 위험도 산정 로직**뿐이다.
- HAZOP처럼 구조가 특수한 기법은 시트 레이아웃만 전용으로 설계한다.
- 위험도 매트릭스·허용기준은 **회사 표준(전 기법 공통 4×4, 5단계 등급 — §5.2)**을 디폴트로 탑재하되, 관리자가 수정할 수 있는 **설정형**으로 구현한다.

## 4. 정보 구조 (조직·데이터 스코프)

### 4.1 조직 계층

```
회사(자사)
 ├─ 본사 (통합 관제)
 ├─ 사업장/현장 #1 (제조 공장 또는 건설 현장)
 │   ├─ 부서/공정 #1-1
 │   │   └─ 작업(Job) / 설비 / 화학물질
 │   └─ 부서/공정 #1-2
 ├─ 사업장/현장 #2 ...
 └─ 협력사(수급업체) A, B, C ...
     └─ 협력사–사업장 계약 관계 (어느 사업장에서 어떤 작업을 수행하는지)
```

- **사업장/현장**: 평가·결재·일정 관리의 기본 단위. 제조업 공장과 건설업 현장을 동일 개념으로 다루되, 유형 속성(제조/건설)을 가진다. 건설 현장은 개설일·준공예정일(현장 수명주기)을 가진다.
- **협력사**: 회사 레벨에 마스터로 등록하고, 사업장별 **계약(투입) 관계**로 연결한다. 하나의 협력사가 여러 사업장에 투입될 수 있다.
- **작업(Job)/설비/화학물질**: 평가 대상 마스터. JSA는 작업, HAZOP은 공정/설비(노드), CHARM은 화학물질을 평가 대상으로 참조한다.

### 4.2 평가서의 스코프 및 용도 (회사 표준)

| 기법 | 용도 (회사 규정) | 평가 대상 단위 | 주 작성 주체 | 실시 유형 연계 |
|---|---|---|---|---|
| **HAZOP** | 산업안전보건법상 **공정안전자료(PSM)**에 포함되는 공정위험성평가 | 사업장 > 공정 > 검토구간(Node) | 자사 공정담당 + 평가팀(스터디) | 최초 / 정기(공정안전보고서 주기 갱신) |
| **KRAS** | **정기 사업장 위험성평가** — 부서·공정·작업 전반을 주기적으로 재점검 | 사업장 > 부서/공정 > 작업·설비 | 자사 부서담당 | 정기(연 1회) / 상시(월 사이클) |
| **JSA** | **신규 작업 발생 시** 작업 착수 전 실시하는 작업위험성평가 | 사업장 > 작업(Job) 단위 | 자사 부서담당 / **협력사 관리자** | **수시**(신규 작업 도입 트리거와 자동 연계 — §5.6 FR-SCHD-003) |
| **CHARM** | 사업장이 **저장·취급하는 화학물질**에 대한 물질위험성평가 | 사업장 > 부서 > 취급 화학물질 | 자사 보건담당 | 정기 / 수시(신규 물질 도입 시) |

> 4개 기법은 용도(공정/정기/신규작업/화학물질)가 서로 달라 병행 운영되며, 하나가 다른 것을 대체하지 않는다. 다만 위험성 결정에는 **공통 4×4 매트릭스**(§5.2)를 사용해 기법 간 위험도 수치를 조직 전체에서 동일한 기준으로 비교할 수 있게 한다.

#### 5.2.1 회사 표준 위험성 결정 매트릭스 (전 기법 공통)

**4×4 매트릭스** (가능성 1~4 × 중대성 1~4, 위험성 = 두 축의 곱, 범위 1~16)

| 위험성 점수 | 등급 | 조치 요구사항 |
|---|---|---|
| **13~16** | 매우 위험 | **즉시** 개선조치 수립 및 시행 |
| **10~12** | 위험 | **빠른 시일 내에** 개선조치 수립 및 시행 |
| **7~9** | 다소 위험 | **1개월 내에** 개선조치 수립 및 시행 |
| **5~6** | 위험성 존재 | 개선조치 수립 및 시행 **필요** |
| **1~4** | 허용 가능한 위험 | 개선조치 수립 및 시행 **불필요** |

- **판정 기준선**: 위험성 **5점 이상**인 모든 요인은 감소대책(개선조치)이 **필수**이며, 미입력 시 결재 상신이 차단된다(FR-ASMT-003 연동).
- 이 매트릭스는 KRAS·JSA·HAZOP·CHARM **4개 기법에 공통 적용**되는 회사 표준이다. 기법별로 두 축의 의미만 다르다: KRAS·JSA·HAZOP은 (발생빈도×발생강도), CHARM은 (유해성 등급×노출수준 등급). 축의 명칭·정의문은 §5.4의 기법별 시트에서 정의한다.
- 관리자는 등급 구간·조치기한·색상을 FR-CFG-002/004에 따라 조정할 수 있으나, 변경 시 버전 관리(FR-CFG-003)가 적용된다.

#### 5.2.2 위험도 축 정의문 디폴트 (절차서 반영)

관리자가 수정 가능한 축 단계 "정의문"의 **시스템 디폴트 문안**으로 절차서 별표1.1을 탑재한다(4×4·곱셈 산정은 §5.2.1과 동일).

**가능성(발생빈도) 1~4 디폴트 정의** — KRAS·JSA·HAZOP 공통:

| 값 | 구분 | 정의(요지) |
|---|---|---|
| 4 | 매우 높음 | 주 1회 정도 노출(빈번), 방호장치 없음, 안전수칙·작업표준이 모호하고 지키기 어려움 |
| 3 | 높음 | 월 1회 정도 노출(가끔), 방호장치는 있으나 미흡, 안전수칙 일부 준수 곤란 |
| 2 | 보통 | 3개월 1회 정도 노출(낮음), 방호·안전장치 설치되어 있으나 위험영역 출입 미제한 |
| 1 | 낮음 | 연 1회 정도 노출(거의 없음), 방호·안전장치 설치 및 위험영역 출입 제한, 수칙 정비·준수 가능 |

**중대성(발생강도) 1~4 디폴트 정의** — 인적 피해 + **재산손실 금액 기준 병기**(절차서 특유):

| 값 | 구분 | 정의(요지) |
|---|---|---|
| 4 | 사망 또는 영구장애 | 사망, 부상 2명 이상, 영구장애 / 재산손실 10억원 초과 |
| 3 | 중상 | 4일 이상 치료 인명사고, 휴업 수반 중대 부상·질병(완치 가능) / 재산손실 1억원 초과 10억원 이하 |
| 2 | 경상 | 1~3일 치료 인명사고, 휴업 없는 부상·질병 / 재산손실 500만원 초과 1억원 이하 |
| 1 | 미미한 재해 | 경미한 인명사고, 치료 후 즉시 작업 가능 / 재산손실 500만원 이하 |

- 등급 표기에 **로마자 병기 옵션**(Ⅴ 매우위험 13~16 / Ⅳ 위험 10~12 / Ⅲ 다소위험 7~9 / Ⅱ 위험성존재 5~6 / Ⅰ 허용가능 1~4)을 제공한다(절차서 별표1.1 <표4>의 표기 관행).
- CHARM(유해성×노출수준)의 축 정의문은 §5.4.4 보강 사항을 따른다.

### 5.4 기법별 평가 시트 (ASMT-H/K/J/C)

#### 5.4.1 KRAS — 빈도·강도법 정기 위험성평가 (ASMT-K)

**용도**: 정기 사업장 위험성평가(연 1회, 상시평가 모드 시 월 사이클). 부서/공정의 작업·설비 전반을 대상으로 한다.

**행 필드 구성** (§5.2.1 회사 표준 4×4 매트릭스 적용)

| 열 | 필드 | 입력 방식 |
|---|---|---|
| 1 | 공정/작업명 | 작업 마스터 선택 또는 직접 입력 |
| 2 | 세부 작업내용 | 텍스트 |
| 3 | 유해·위험요인 분류 | 코드 선택(기계적/전기적/화학적/생물학적/작업특성/작업환경) |
| 4 | 유해·위험요인 및 발생상황 | 텍스트 (원인과 결과가 드러나게 서술) |
| 5 | 관련 법규·근거 | 선택 입력 |
| 6 | 현재 안전보건조치 | 텍스트 |
| 7 | 발생빈도(가능성) | 1–4 선택 |
| 8 | 발생강도(중대성) | 1–4 선택 |
| 9 | 위험성 | 자동: 빈도×강도 (1–16), §5.2.1 5단계 등급·색상 표시 |
| 10 | 개선조치 필요 여부 | 자동 판정 (5점 이상 = 필요, 1~4 = 불필요) |
| 11 | 감소대책 (유형분류 필수) | 5점 이상 시 필수 |
| 12 | 개선 후 위험성 | 빈도·강도 재선택 → 자동 계산 |
| 13 | 조치 담당자/기한 | 사용자·날짜 선택 (§5.2.1 등급별 기한 디폴트 자동 제안) |

- FR-ASMT-K-002 (P1): 부서/공정 선택 시 해당 조직의 작업 마스터 목록을 불러와 행을 일괄 생성할 수 있다.

**보강 사항(절차서 반영)**

- 행 필드에 **비고(재해이력 태그: 아차사고/산업재해/동종사고)** 추가(→ FR-ASMT-013).
- 행 필드에 **개선(완료)일** 추가(조치 과제(FR-ACT)의 완료일과 동기화 표시).
- 유해·위험요인 분류 코드 마스터(FR-CFG-005)의 디폴트 값으로 절차서 별표2.2 분류표(기계적/전기적/화학물질/생물학적/작업특성/작업환경 6대분류 및 세부항목: 끼임, 위험한 표면, 낙하·비래·전복, 충돌, 넘어짐, 추락(개구부), 감전, 아크, 정전기, 화재/폭발, 가스·증기·에어로졸·흄·미스트·분진, 반응성 물질, 방사선, 병원성 미생물, GMO, 소음, 진동, 휴먼에러, 질식·산소결핍, 중량물 취급, 반복작업, 불안정한 작업자세, 조명, 공간·이동통로, 조직 안전문화 등)를 탑재한다 — KOSHA 표준모델 기준 대분류 6·소분류 43.

#### 5.4.2 JSA — 작업안전분석 (ASMT-J)

**용도**: **신규 작업 발생 시** 작업 착수 전 실시하는 수시평가. 작업(Job) 하나를 단계(Step)로 분해하여 평가한다. **협력사 관리자가 작성·제출하는 기본 기법**이다.

**구조**: 평가서(작업 1건) > 작업단계(Step, 순번) > 단계별 요인 행(복수)

**작업단계 필드**: 순번, 단계명(동사형 서술), 단계 설명, 사용 장비·공구

**단계별 요인 행 필드** (§5.2.1 회사 표준 4×4 매트릭스 적용)

| 열 | 필드 | 입력 방식 |
|---|---|---|
| 1 | 유해·위험요인 | 분류코드 + 서술 |
| 2 | 재해 형태 | KOSHA 재해분류 선택 |
| 3 | 현재 안전조치 | 텍스트 |
| 4 | 발생빈도 | 1–4 선택 |
| 5 | 발생강도 | 1–4 선택 |
| 6 | 위험성 | 자동: 빈도×강도 (1–16), §5.2.1 5단계 등급 표시 |
| 7 | 감소대책(유형분류, 5점 이상 필수) / 개선 후 위험성 / 담당·기한 | 공통 규칙 동일 |

**JSA 헤더 추가 필드**: 작업 위치, 작업 예정일시, 필요 자격·면허, 필요 보호구 체크리스트, 관련 작업허가 여부(참조 텍스트), **필요 자료(작업계획서, 작업허가서, MSDS, 도면 P&ID·전기단선도 등), 필요 안전장비(가스감지기, 소화기 등), 사용 물질, 작업번호, 개정일자**(절차서 반영).

**보강 사항(절차서 반영)**

- **작업단계 중분류(그룹) 지원**: 작업단계가 10개를 초과할 경우 중분류(예: 분리작업/운반작업/청소작업/조립작업)를 만들고 그 하위에 단계를 배치할 수 있다. S-13 좌측 트리에 2계층 표시.
- **단계 구분 입력 가이드**(FR-ASMT-011 입력 가이드라인에 반영): "단계는 10개 내외 권장", "행동 중심 단어가 마지막에 오는 명칭(예: 덮개 운반, 볼트 조립)", "너무 세분·포괄 금지", "특별한 위험이 없는 단계는 병합 가능" — 초과·미달 시 경고성 안내 표시.
- **유해위험요인 파악용 점검표(22항목) 제공**: 요인 입력 보조 패널로 절차서 별표3.1의 22개 질문(작업위치/통행로/고온/제한공간/고소/도구·장비/소음·진동/끼임·감김/화학물질/독성가스/전기/추락/위험에너지원/자동 가동·정지/작업자 적합성/중량물/보호구/에너지 차단·잠금(LOTO)/비상조치계획 등)을 체크리스트로 띄우고, 해당 항목에서 요인 행 생성으로 연결한다.
- **현장확인 연계**: 단계별 요인 행에 현장확인(작업전/작업중 × 작업자/감독자) 상태 표시(→ FR-ASMT-J-005).

- FR-ASMT-J-001 (P0): 단계 추가·삭제·순서 변경(드래그)이 가능하다.
- FR-ASMT-J-002 (P0): 협력사 관리자가 작성하는 JSA는 §5.5 협력사 제출 플로우를 따른다.
- FR-ASMT-J-003 (P0): JSA는 **신규 작업(수시평가 트리거 사유 ④ 작업방법·절차 신규 도입)과 자동 연계**된다. 수시평가 트리거 이벤트(§5.6 FR-SCHD-003)에서 '신규 작업'을 선택하면 JSA 평가서 생성이 강제되고, 작업 개시 예정일까지 승인이 완료되지 않으면 경보가 발생한다(FR-SCHD-004). 다만 절차서상 최초·정기평가에도 (작업) JSA가 쓰이는 사례가 있으므로, **구현 시 JSA 실시 유형에 최초/정기 선택을 차단하지 않는다**(§5.2.3 참고).
- FR-ASMT-J-004 (P1): 유사 작업의 승인된 JSA를 템플릿으로 복제해 새 작업에 적용할 수 있다. (→ 표준모델 도입 후에는 FR-ASMT-014로 일원화 검토)
- FR-ASMT-J-005 (P1): **작업 당일 현장확인 체크**(절차서 반영): 승인 완료된 JSA에 대해 작업일 단위로 '현장확인 세션'을 생성하고, 작업단계별로 **작업 전 / 작업 중** 각각 작업자(V)·감독자(O)가 조치 이행 여부를 체크한다(모바일 우선). 체크자·일시가 기록되며, 현장 확인 중 추가 위험 발견 시 해당 JSA의 재평가(개정)로 연결하는 버튼을 제공한다. 세션에는 "TBM 등을 통한 참여 인원 공유 확인" 체크 항목을 포함한다(TBM 회의 관리 자체는 기존 범위 제외 유지, 확인 체크만 지원).
- FR-ASMT-J-006 (P2): **현장 비치용 출력**(절차서 반영): JSA 승인본을 안전작업허가서 첨부·현장 비치 용도의 1~2장 요약 양식(작업단계·요인·대책·현장확인 체크란 포함)으로 출력할 수 있다(FR-ASMT-009 확장).

**S-13 JSA 시트 — 화면 설계 상세**

화면은 3단 구성이다.

1. **상단 고정 영역(요약바)**: 평가서 번호, 작업명, 실시유형(수시) 배지, 상태(작성중/결재중/승인완료) 배지, 저장 버튼(자동저장 + 수동저장), "결재 상신" 버튼(위험성 5점 이상 항목에 감소대책이 비어 있으면 비활성화되고 사유 툴팁 표시).
2. **좌측 패널 — 작업단계 트리**: 세로 스텝 리스트(①②③…). 각 스텝 카드는 단계명 요약과 해당 스텝 내 최고 위험등급 색상 배지를 보여준다(요인 미입력 스텝은 회색 "미평가" 배지). 드래그 핸들로 순서 변경, 하단 "+ 단계 추가" 버튼. 스텝을 클릭하면 우측 패널이 해당 스텝으로 전환된다.
3. **우측(메인) 패널 — 선택된 단계 상세**
   - 상단: 단계명(동사형 서술 placeholder 안내, 예: "비계를 설치한다"), 단계 설명, 사용 장비·공구(태그 입력).
   - 하단: 유해·위험요인 행 테이블(열 1~7). 행 추가 시 인라인 아코디언으로 확장. 빈도×강도는 숫자 드롭다운 대신 **4×4 그리드 팝오버**(가로=빈도, 세로=강도, 셀 클릭으로 즉시 값 반영)로 선택하며, 계산된 위험성 점수는 §5.2.1 색상 배지로 즉시 표시된다.
   - 위험성 5점 이상 행은 감소대책 입력란이 붉은 테두리 + 필수(*) 표시로 강조된다. 유형(제거·대체/공학적/관리적/개인보호구) 미선택 상태로 임시저장은 가능하나 결재 상신은 차단된다.
4. **헤더 폼 진입점**: 상단 "작업 정보" 탭에서 별도 패널로 열리며, 공통 헤더(§5.3) 아래 JSA 전용 필드(작업 위치, 작업 예정일시, 필요 자격·면허, 필요 보호구 체크리스트, 관련 작업허가 여부)를 이어서 배치한다.
5. **모바일(협력사 관리자 P5) 대응**: 좌측 트리 + 우측 상세의 2단 레이아웃 대신, 스텝을 아코디언 카드로 세로 나열하는 1단 레이아웃으로 전환한다. 빈도×강도 선택은 풀스크린 4×4 그리드 모달로 표시하고, 요인 행 하단 카메라 아이콘으로 즉시 촬영·업로드한다(FR-ASMT-008).
6. **트리거 연계 배너**: 수시평가 트리거(FR-ASMT-J-003)로 생성된 JSA는 화면 상단에 "신규 작업 트리거: [트리거명] · 작업 개시 예정일 D-n" 배너를 표시하고, 클릭 시 트리거 등록 화면(S-21)으로 이동한다. D-day 이내 미승인 시 배너가 경고색으로 전환된다.
7. **중분류 트리(절차서 반영)**: 좌측 작업단계 트리는 단계가 10개를 초과하면 중분류(그룹) 노드 아래 단계를 배치하는 2계층 트리로 전환된다. 각 요인 행에는 현장확인(작업전/작업중 × 작업자/감독자) 상태 배지가 표시된다(승인완료 이후, FR-ASMT-J-005 세션 생성 시).

#### 5.4.3 HAZOP — 공정 위험성평가 (ASMT-H)

**용도**: 산업안전보건법상 **공정안전자료(PSM, Process Safety Management)**에 포함되는 공정위험성평가. 연속 공정(화학·유틸리티 설비 등)을 검토구간(Node)과 가이드워드로 체계 분석한다. 팀 검토 회의 기반.

**구조**: 평가서(공정/스터디 1건) > 검토구간(Node, 복수) > 이탈(Deviation) 행(복수)

**평가서 헤더 추가 필드**: 대상 공정명, 참조 도면(P&ID 번호·파일 첨부), 검토 회의 일자(복수 세션), 스터디 리더·서기 지정, **공정안전보고서(PSM) 문서번호**(해당 시 연결)

**검토구간(Node) 필드**: 노드 번호, 구간 설명(시작점~끝점), 설계 의도(Design Intent: 유량·압력·온도 등 정상 운전 조건)

**이탈 행 필드** (§5.2.1 회사 표준 4×4 매트릭스 적용)

| 열 | 필드 | 입력 방식 |
|---|---|---|
| 1 | 공정 변수 | 선택: 유량, 압력, 온도, 액위, 조성, 반응, 시간, 기타 |
| 2 | 가이드워드 | 선택: No/Not, More, Less, Reverse, Part of, As well as, Other than |
| 3 | 이탈(Deviation) | 자동 조합 + 서술 보완 (예: "유량 없음 — No Flow") |
| 4 | 원인(Cause) | 텍스트, 복수 등록 |
| 5 | 결과(Consequence) | 텍스트 |
| 6 | 현재 안전장치(Safeguard) | 텍스트 (인터록, 알람, PSV 등) |
| 7 | 발생빈도 | 1–4 선택 |
| 8 | 발생강도 | 1–4 선택 |
| 9 | 위험성 | 자동: 빈도×강도 (1–16), §5.2.1 5단계 등급 판정 |
| 10 | 개선권고사항(Recommendation) | 5점 이상 시 필수, 대책 유형분류 |
| 11 | 조치 담당자/기한 | 공통 규칙 동일 |

- FR-ASMT-H-001 (P0): 노드·이탈 행의 트리형 입력과 노드별 진행률 표시.
- FR-ASMT-H-002 (P0): 평가서를 **공정안전자료(PSM) 문서 세트의 첨부물**로 출력할 수 있는 전용 양식(FR-ASMT-009 확장)을 제공한다.
- FR-ASMT-H-003 (P1): 가이드워드×공정변수 조합 매트릭스에서 검토 완료 여부를 체크해 누락 조합을 확인할 수 있다.
- FR-ASMT-H-004 (P2): 개선권고사항(Recommendation) 목록만 별도 시트로 추출·출력(HAZOP Close-out 관리). **개선권고사항 필드에 우선순위, 책임부서, 검토 시나리오 요지(무슨 조치가/어디에/왜 필요한가)를 추가**(절차서 별표6.7 감소대책 조치계획 양식 대응).

**보강 사항(절차서 반영)**

- **사전조사 자료 카테고리 디폴트**(헤더 첨부 분류): 설계 개념, 공정흐름도(PFD)·물질/열수지, 주요 기계장치 설계자료, 공정 설명서·제어계통 설명서, 설비 배치도면, P&ID, 정상·비정상 운전절차, **경보·자동운전정지(Trip) 설정치 목록**, MSDS, 안전밸브 설정치·용량 산출자료, 배관 표준·명세, 과거 중대산업사고·공정사고·아차사고 사례.
- **검토구간(Node) 설정 가이드**(입력 가이드라인, FR-ASMT-011 반영): 공정흐름 순서를 따를 것, 원료 투입 배관 주변을 첫 구간으로, 설계목적·운전조건 변경 지점 및 후속 설비 연결 지점에서 구간 분리, 도면이 바뀌어도 배관이 연결되면 동일 구간.
- **이탈·원인 예시 라이브러리 디폴트 탑재**: 절차서 별표6.5(변수별 이탈 및 가능한 원인 — 유량/액면/압력/온도/혼합/반응/점도/압력방출/계측/시료채취/부식·침식/서비스 실패/비정상 운전/유지보수/점화원·정전기/예비장비/안전)를 참조 데이터로 제공하고, 이탈 행 입력 시 원인 후보를 제안한다.
- 이탈 행 **비고(재해이력 태그)** 추가(→ FR-ASMT-013).

#### 5.4.4 CHARM — 화학물질 위험성평가 (ASMT-C)

**용도**: 사업장이 **저장·취급하는 화학물질**에 대한 물질위험성평가. 취급 화학물질별로 유해성 등급 × 노출수준 등급으로 위험성을 산정한다(KOSHA CHARM 기법 기반).

**구조**: 평가서(부서/공정 1건) > 화학물질 행(복수)

**화학물질 행 필드** (§5.2.1 회사 표준 4×4 매트릭스 적용 — 축은 유해성×노출수준)

| 열 | 필드 | 입력 방식 |
|---|---|---|
| 1 | 화학물질명 / CAS No. | 화학물질 마스터 선택(MSDS 첨부 연동) |
| 2 | 취급 공정·용도 | 텍스트 |
| 3 | 유해성 등급 (1–4) | 두 가지 산정 경로 지원: ① 노출기준(TWA) 기반 자동 제안 ② GHS 분류(H-code, CMR 여부) 기반 자동 제안. CMR(발암성 1A/1B 등) 물질은 자동으로 최고 등급(4). |
| 4 | 노출수준 등급 (1–4) | 두 가지 산정 경로 지원: ① 작업환경측정 결과 입력 시 측정치/노출기준 비율로 자동 산정 ② 측정 결과 없으면 하루 취급량(대/중/소) × 비산성·휘발성(고/중/저) 매트릭스로 산정 |
| 5 | 위험성 | 자동: 유해성 × 노출수준 (1–16), §5.2.1 5단계 등급 판정 |
| 6 | 현재 관리 조치 | 국소배기, 밀폐, 보호구 등 텍스트 |
| 7 | 감소대책(5점 이상 필수) / 개선 후 등급 / 담당·기한 | 공통 규칙 동일 |

- FR-ASMT-C-001 (P0): 화학물질 마스터(물질명, CAS, 노출기준, GHS 분류, MSDS 파일)를 사업장별로 등록·관리할 수 있다.
- FR-ASMT-C-002 (P1): 유해성·노출수준 등급의 자동 제안 값을 평가자가 근거 입력과 함께 수동 조정할 수 있다(조정 이력 기록).
- FR-ASMT-C-003 (P2): 작업환경측정 결과(측정치, 측정일)를 물질·공정별로 입력·이력 관리하고 노출수준 산정에 재사용한다.

**보강 사항(절차서 반영) — 산정 규칙 정밀화**

FR-ASMT-C-002의 "자동 제안"에 다음 절차서 규칙을 디폴트 산정 로직으로 명시한다.

**노출수준(가능성) 산정**

1. 작업환경측정 결과가 있는 경우(별표5.4): 노출수준(%) = 측정결과/노출기준 × 100 → **100% 초과=4 / 50~100%=3 / 10~50%=2 / 10% 이하=1**.
2. 측정 결과가 없는 경우(별표5.5): 하루 취급량(3=ton·㎥ / 2=kg·ℓ / 1=g·㎖) × 비산성(분진·흄: 고3/중2/저1) 또는 휘발성(가스·증기: 끓는점 기준 — 상온 시 끓는점<50℃=고, 50~150℃=중, >150℃=저; 공정온도 X℃ 시 <2X+10=고, 2X+10~5X+50=중, >5X+50=저) 조합표로 1차 산정 후, **밀폐·환기상태 보정을 차감**한다(원격조작·완전밀폐=−2, 국소배기장치 설치=−1).
3. **직업병 유소견자가 발생한 물질은 측정결과·취급량과 무관하게 노출수준 = 4 강제**.

**유해성(중대성) 산정**

1. 노출기준 설정 물질(별표5.6): 분진 0.01㎎/㎥ 이하=4 / 0.01~0.1=3 / 0.1~1=2 / 1~10=1, 증기 0.5ppm 이하=4 / 0.5~5=3 / 5~50=2 / 50~500=1.
2. 노출기준 미설정 또는 초과(분진 10㎎/㎥·증기 500ppm 초과) 물질(별표5.7): MSDS의 **H-code(유해·위험문구)/R-phrase 매핑표**로 결정(예: H350·H340·H330=4, H331·H314·H317=3, H332·H302=2, H319·H315=1, 미분류 기타=1). 매핑표를 코드 마스터로 탑재한다.
3. **CMR(발암성·생식세포 변이원성·생식독성) 1A/1B/2 물질은 노출기준과 무관하게 유해성 = 4**(2 구분 포함을 명확화).

**공통**

- **혼합물질은 구성 단일물질별 위험성 계산 결과 중 최고값을 그 혼합물질의 위험성으로 결정**한다.
- 행에 산정 근거 스냅숏(유소견자 여부, 측정결과, 하루 취급량, 비산/휘발성, 밀폐·환기 보정, CMR 여부, 노출기준, H-code)을 저장·표시한다.
- 화학물질 마스터(FR-ASMT-C-001) 필드 보강: 제조/사용 구분, 사용 용도, **월 취급량, 직업병 유소견자 발생 여부**, MSDS 보유 여부. MSDS 참조 위치 안내(노출기준→§8, 물성→§9, CMR→§11, H-code→§2)를 도움말로 제공.

### 5.5 전자결재 및 협력사 제출 플로우 (APRV)

| ID | 우선순위 | 요구사항 |
|---|---|---|
| FR-APRV-001 | P0 | 결재선은 **작성 → 검토 → 승인** 3단계를 기본으로 한다. 검토자·승인자는 결재선 템플릿(FR-CFG-006)에서 자동 지정되며 상신 시 변경 가능하다. 검토 단계는 복수 지정(순차) 가능, 생략도 설정 가능. |
| FR-APRV-002 | P0 | 각 결재 단계에서 승인/반려를 선택할 수 있고, 반려 시 **반려 사유 입력이 필수**다. 반려된 평가서는 작성중 상태로 돌아가고 작성자에게 알림이 간다. |
| FR-APRV-003 | P0 | 결재 이력(단계, 처리자, 처리일시, 의견)은 평가서에 영구 기록되고 출력물에 포함된다. |
| FR-APRV-004 | P0 | **협력사 제출 플로우**: 협력사 관리자가 작성한 평가서(주로 JSA)는 '제출' 시 자사 측 결재선(검토: 담당 부서, 승인: 사업장 안전관리자)으로 상신된다. 도급인의 수급인 평가 검토 의무 이행 기록이 된다. |
| FR-APRV-005 | P0 | 반려된 협력사 평가서는 협력사 관리자에게 반려 사유와 함께 반환되고, 수정 후 재제출하면 재제출 회차가 기록된다. |
| FR-APRV-006 | P1 | 위험등급별 승인권자 상향 규칙: **'매우 위험'(13~16점)** 등급 항목이 포함된 평가서는 설정된 상위 승인자(예: 본사 총괄)의 승인을 추가로 요구할 수 있다(FR-CFG-004 연동). |
| FR-APRV-007 | P1 | 결재 대기함/처리함: 내가 처리할 결재, 내가 상신한 문서의 진행 상태를 한 화면에서 본다. 모바일에서 승인·반려 처리 가능. |
| FR-APRV-008 | P2 | 대결(대리결재) 지정: 승인자 부재 시 기간을 정해 대결자를 지정할 수 있다(대결 표시 기록). |
| FR-APRV-009 | P1 | **수급업체 위험성평가 이행점검 체크리스트**(절차서 반영): 협력사 제출 평가서의 검토 단계에서 정형 점검표를 작성한다 — ① 위험성 판단기준 보유 여부 ② 위험도(중대성) 반영 내용의 적정성 ③ 판단기준에 따른 감소대책 수립 항목의 적용 여부 ④ 평가 시 해당 작업 근로자의 참여·공유 여부 + 종합 의견(사전 제공 자료, 점검·검토 결과, 보완 조치 필요사항). 검토자·확인자가 각각 서명하며, 점검표는 평가서에 첨부·보존되고 승인/반려 판단의 근거로 출력물에 포함된다. |

### 5.6 평가 일정·트리거 관리 (SCHD)

> 고시 제15조: 최초(성립 후 1개월 내 착수), 수시(사유 발생 시 작업 개시 전), 정기(매 1년), 상시(월·주·일 — 이행 시 수시+정기 갈음).

| ID | 우선순위 | 요구사항 |
|---|---|---|
| FR-SCHD-001 | P0 | 사업장별 평가 운영 모드를 선택한다: **「정기+수시」 모드** 또는 **「상시평가」 모드**. |
| FR-SCHD-002 | P0 | **정기평가 일정**: 사업장·부서별 최초/직전 정기평가 완료일 기준으로 차기 도래일(1년)을 자동 산출하고, D-60/D-30/D-7/기한 초과 시점에 담당자·안전관리자에게 알림을 발송한다. 기한 초과 건은 대시보드에 경보로 표시된다. |
| FR-SCHD-003 | P0 | **수시평가 트리거 이벤트 등록**: 사유 6종을 선택해 이벤트를 등록한다 — ① 건설물 설치·이전·변경·해체 ② 기계·기구·설비·원재료 신규 도입/변경 ③ 건설물·설비 정비·보수 ④ 작업방법·절차 신규 도입/변경 ⑤ 산업재해·아차사고 발생 ⑥ 중대산업사고 발생. 이벤트에는 대상 작업 개시 예정일을 입력한다. |
| FR-SCHD-004 | P0 | 트리거 이벤트에는 수시평가서가 연결되어야 하며, **작업 개시 예정일까지 승인 완료되지 않으면** 경보를 발생시킨다(수시평가는 작업 개시 전 실시 의무). 재해 사유(⑤⑥)는 재발방지대책 수립 전 실시 원칙을 안내 문구로 표시한다. |
| FR-SCHD-005 | P0 | **상시평가 모드 — 월 단위**: 월별로 '월 위험성평가(순회점검·근로자 제안·아차사고 확인 결과 반영)'를 1건 이상 완료해야 한다. 미완료 월은 경보 표시. 근로자 제보(FR-SHRE-005)와 아차사고 이벤트를 월 평가서에 연결할 수 있다. |
| FR-SCHD-006 | P0 | **상시평가 모드 — 주 단위**: 주별로 안전관리자 등이 '주간 결과 공유·이행상황 점검' 기록(점검 항목 체크 + 특이사항)을 등록해야 한다. 미등록 주는 경보 표시. |
| FR-SCHD-007 | P0 | **상시평가 이행 판정**: 월·주 사이클의 이행 여부를 월 단위로 자동 판정해 '갈음 요건 충족/미충족'을 대시보드에 표시한다. ※ 일 단위 주지(TBM)는 시스템 범위 외 — 평가 결과의 상시 공유·열람 기능(§5.8)으로 근로자 주지를 지원하되, 일 단위 이행 판정에는 포함하지 않고 사업장 자체 운영 사항으로 안내한다. |
| FR-SCHD-008 | P1 | **최초평가 관리**: 신규 사업장(현장) 등록 시 개설일 기준 1개월 카운트다운을 표시하고 최초평가서 연결 여부를 추적한다. |
| FR-SCHD-009 | P1 | 사업장 평가 캘린더: 정기 도래일, 수시 이벤트, 상시 월·주 사이클, 조치 기한을 월간 캘린더 뷰로 제공한다. |
| FR-SCHD-010 | P1 | **연간 정기평가 실시계획 수립**(절차서 반영): 사업장 단위로 연간 실시계획(평가 일정, 대상 부서/공정, 적용 기법, 평가 인원, **평가 전 교육 실시 방안**, 기타)을 등록하고 결재(승인권자: 사업장 안전관리자 → 안전보건관리책임자)를 받는다. 승인된 계획은 FR-SCHD-002 정기 도래일 관리와 연결된다. |
| FR-SCHD-011 | P2 | **본사 계획 전개**(절차서 반영): 본사 총괄이 전사 실시계획(기준 일정·중점 사항)을 등록해 전 사업장에 배포하고, 각 사업장 계획의 수립·승인 현황을 본사 대시보드에서 추적한다(절차서상 5월 전사 계획 → 6월 사업장 계획 순서. 사업장 계획 先수립 후 본사 반영도 허용). |
| FR-SCHD-012 | P1 | **미조치 사항 차년도 이월**(절차서 반영): 연말(또는 계획 수립 시점) 기준 미완료 개선조치를 조회해 차년도 실시계획의 '전년도 미조치 반영' 목록으로 일괄 이월·연결할 수 있다. |
| FR-SCHD-013 | P1 | **수시평가 이행점검 체크리스트**(절차서 반영): 반기 1회 이상(정기평가 시점 포함), 기간 내 실시된 수시평가서 목록을 불러와 "작업내용·유해위험요인의 표준모델 반영 필요 여부"를 점검하고 결과(해당 표준모델명, 반영 내역: 추가/변경/삭제)를 기록한다. 작성자(차·팀장급)와 확인자(부장급)의 확인 서명이 남고, 반영 필요 항목은 FR-ASMT-014 표준모델 개정 작업과 연결된다. 반기 내 미실시 시 경보를 표시한다. |

### 5.7 개선조치 이행관리 (ACT)

| ID | 우선순위 | 요구사항 |
|---|---|---|
| FR-ACT-001 | P0 | 평가서 승인 시, 감소대책이 입력된 각 행은 자동으로 **개선조치 과제**로 생성된다(담당자, 기한, 대책 내용, 원 평가서·행 링크 포함). |
| FR-ACT-002 | P0 | 조치 담당자는 '내 조치 목록'에서 과제를 확인하고 진행 상태를 갱신한다: 예정 → 진행중 → 완료요청 → 확인완료. |
| FR-ACT-003 | P0 | **완료요청 시 증빙 사진 첨부 필수**(개선 전/후 사진 권장 2매). 모바일 카메라 직접 촬영 업로드를 지원한다. |
| FR-ACT-004 | P0 | 완료요청은 확인권자(디폴트: 사업장 안전관리자)가 **확인 승인**해야 확인완료로 전환된다. 미흡 시 보완요청(사유 입력)으로 반려할 수 있다. |
| FR-ACT-005 | P0 | 확인완료 시 **개선 후 위험성 재평가**를 입력한다(기법별 축 값 재선택 → 자동 산정). 재평가 결과가 여전히 **5점 이상**이면 추가 대책 등록을 요구한다(고시 제12조: 조치 후 잔여 위험성 확인). |
| FR-ACT-006 | P0 | 기한 초과 조치는 담당자·안전관리자에게 알림 + 대시보드 경보로 표시된다. |
| FR-ACT-007 | P1 | 조치 기한 연장 요청·승인 플로우(사유 필수, 이력 기록). |
| FR-ACT-008 | P1 | 협력사 평가서에서 생성된 조치는 협력사 관리자가 처리하고 자사 안전관리자가 확인한다. |
| FR-ACT-009 | P2 | 조치 유형·비용 태그를 입력해 개선 투자 통계에 활용한다. |

### 5.8 결과 공유·열람 서명 (SHRE)

> 법 제36조 2항(근로자 참여), 고시 제13조(결과 공유). 참여·공유 대상은 고용형태·국적 불문 사업장의 모든 근로자.

| ID | 우선순위 | 요구사항 |
|---|---|---|
| FR-SHRE-001 | P0 | 승인 완료된 평가서를 대상 근로자 그룹(부서, 작업, 협력사 단위 선택)에 **공유 발행**할 수 있다. 공유 시 대상자 전원에게 시스템 알림이 발송된다. |
| FR-SHRE-002 | P0 | 공유 화면(모바일 최적화)은 요약 뷰를 제공한다: 대상 작업, 주요 유해·위험요인, 위험등급, 감소대책, 근로자 준수사항. 원문 평가서도 열람 가능. |
| FR-SHRE-003 | P0 | 근로자는 열람 후 **전자서명(터치 서명 또는 확인 버튼+본인인증)**으로 열람 확인을 남긴다. 서명 일시·서명자·대상 평가서 버전이 기록된다. |
| FR-SHRE-004 | P0 | 평가서별 서명 현황(대상자 수, 서명 완료 수, 미서명자 명단)을 관리자가 조회하고 미서명자에게 재알림을 보낼 수 있다. |
| FR-SHRE-005 | P1 | **위험요인 제보**: 모든 근로자(협력사 포함)는 모바일에서 사진+텍스트로 위험요인·아차사고를 제보할 수 있다. 제보는 사업장 안전관리자에게 전달되고, 처리 상태(접수→검토→평가반영/조치/기각)와 처리 결과가 제보자에게 회신된다. 제보는 수시평가 트리거 또는 상시평가 월 평가서에 연결할 수 있다. |
| FR-SHRE-006 | P1 | 사업장 게시판 뷰: 현재 유효한(최신 승인) 평가 결과 목록을 사업장 단위로 상시 게시한다(고시 제13조 게시 의무 대응). |
| FR-SHRE-007 | P2 | QR코드 발행: 작업장소·설비에 부착할 QR을 평가서별로 생성, 스캔 시 해당 공유 화면으로 이동(로그인 필요). |
| FR-SHRE-008 | P1 | **도급 안전보건정보 제공**(절차서 반영): 협력사–사업장 계약(FR-ORG-006) 등록·갱신 시, 도급인 측이 해당 협력사에 제공할 안전보건정보 패키지를 구성·발행할 수 있다 — ① 해당 사업장·작업 관련 위험성평가 결과(최신 승인본) ② 유해위험물질 정보·MSDS ③ 작업환경측정 결과 ④ 비상시 경보체계·대피방법 ⑤ 기타. 협력사 관리자는 자기 업체에 발행된 정보를 열람·다운로드하고 열람 확인이 기록된다. 제공 이력(일시·항목·수신 업체)은 도급인 의무 이행 증빙으로 보존된다. |

### 5.9 기록·보존 및 감사 (REC)

> 시행규칙 제37조: ① 평가 대상의 유해·위험요인 ② 위험성 결정 내용 ③ 결정에 따른 조치 내용 ④ 사전조사한 안전보건정보 등 — **3년 보존**.

| ID | 우선순위 | 요구사항 |
|---|---|---|
| FR-REC-001 | P0 | 승인 완료된 평가서·결재 이력·서명 기록·조치 이력은 **수정·삭제 불가**로 보존된다. 변경은 개정(새 버전)으로만 가능하다. |

## 8. 데이터 모델 개요 (핵심 엔티티)

상세 스키마는 설계 단계 산출물로 하고, 여기서는 엔티티와 관계만 정의한다.

```
Company ─┬─ Site(사업장/현장) ─┬─ Dept(부서/공정, 트리)
         │                    ├─ Job(작업 마스터)
         │                    ├─ Chemical(화학물질 마스터, MSDS)
         │                    └─ PartnerContract(협력사 투입계약)
         ├─ Partner(협력사) ── PartnerContract
         └─ User(사용자) ── RoleAssignment(역할×스코프)

Assessment(평가서: 공통 헤더, 기법, 실시유형, 상태, 기준버전 스냅숏)
 ├─ AssessmentMember(평가팀·참여 근로자)
 ├─ HazardItem(요인 행: 공통 필드 + 기법별 확장 필드)
 │    ├─ (JSA) JobStep 소속        ├─ (HAZOP) Node·Deviation 소속
 │    └─ Action(개선조치 과제: 담당, 기한, 상태, 증빙, 재평가 결과)
 ├─ ApprovalFlow(결재선) ── ApprovalStep(단계별 처리 이력)
 ├─ SharePublication(공유 발행) ── Acknowledgement(열람 서명)
 └─ Attachment(첨부: 사진, MSDS, 도면)

RiskMatrixConfig(기법별 매트릭스·허용기준, 버전 관리)
ScheduleRule(정기 도래일) / TriggerEvent(수시 사유) / OngoingCycle(상시 월·주 이행 기록)
HazardReport(위험요인 제보) / Notification(알림) / AuditLog(감사 로그)
```

**모델링 원칙**

- `HazardItem`은 공통 컬럼 + 기법별 확장(JSON 또는 확장 테이블)으로 설계해 공통 엔진(결재·조치·공유·통계)이 기법과 무관하게 동작하게 한다.
- 승인 시점에 평가서 전체(적용 매트릭스 버전 포함)를 **스냅숏 보존**하여 이후 설정 변경이 과거 문서에 영향을 주지 않게 한다.
- 모든 도메인 테이블은 사업장(Site) 스코프 키를 가지며, 협력사 데이터는 Partner 키로 추가 격리한다.

### 8.1 JSA 데이터 모델 상세

> §8 본문 엔티티(`Assessment`, `HazardItem` 등)의 JSA 확장을 다룬 논리 데이터 모델이다.

```
Assessment (기법='JSA', 실시유형='수시')
 └─ AssessmentJSA (1:1 확장)
      - 작업위치 (VARCHAR)
      - 작업예정일시 (DATETIME)
      - 필요자격면허 (VARCHAR[] / JSON)
      - 필요보호구체크리스트 (JSON: [{item, required}])
      - 관련작업허가참조 (VARCHAR, nullable)
      - trigger_event_id (FK → TriggerEvent, nullable — 수시 트리거로 생성된 경우)

Assessment(JSA) 1 ── N JobStep
 JobStep
  - id (PK)
  - assessment_id (FK)
  - seq_no (INT, 정렬 순번 — assessment_id 범위 내 유일)
  - step_name (VARCHAR, 동사형 서술)
  - step_desc (TEXT)
  - equipment_tools (VARCHAR[] / JSON)

JobStep 1 ── N HazardItem (JSA 확장)
 HazardItem
  - id (PK)
  - job_step_id (FK)
  - hazard_type_code (FK → 유해·위험요인 분류코드 마스터)
  - hazard_desc (TEXT)
  - injury_type_code (FK → KOSHA 재해분류 마스터)
  - current_measure (TEXT)
  - freq (INT 1–4) / severity (INT 1–4)
  - risk_score (INT, computed = freq × severity)
  - risk_grade (ENUM: 매우위험/위험/다소위험/위험성존재/허용가능, computed)
  - needs_action (BOOLEAN, computed = risk_score ≥ 5)
  - reduction_measure_type (ENUM: 제거대체/공학적/관리적/개인보호구, needs_action=true면 필수)
  - reduction_measure_desc (TEXT, needs_action=true면 필수)
  - residual_freq / residual_severity / residual_risk_score (개선 후 위험성)
  - matrix_version_id (FK → RiskMatrixConfig, 작성 시점 스냅숏)

HazardItem 1 ── 0..1 Action (needs_action=true면 필수 생성)
 Action
  - id (PK)
  - hazard_item_id (FK)
  - owner_user_id (FK → User)
  - due_date (DATE, 등급별 디폴트 자동제안 + 수정가능)
  - status (ENUM: 대기/진행중/완료/지연)
  - evidence_attachment_id (FK → Attachment, nullable)
  - verified_by / verified_at (완료 확인자)

HazardItem 1 ── N Attachment (현장 사진 등)
```

**JSA 고유 제약조건**

- `AssessmentJSA.trigger_event_id`가 존재하면, 해당 Assessment는 `TriggerEvent.target_start_date`(작업 개시 예정일)까지 상태가 '승인완료'로 전이되지 않는 한 배치 잡이 매일 경보(Notification)를 생성한다(FR-SCHD-004).
- `HazardItem.risk_score`가 5 이상인데 `reduction_measure_type`이 NULL이면, Assessment 상태를 '작성중'→'결재중'으로 전이하는 API가 검증 오류를 반환한다(FR-ASMT-003의 서버 측 강제 로직).
- `JobStep.seq_no` 재정렬(드래그)은 assessment_id 범위 내 일괄 갱신(bulk update) 트랜잭션으로 처리해 순번 충돌을 방지한다.

## 12. 용어집

| 용어 | 정의 |
|---|---|
| 위험성평가 | 유해·위험요인을 파악하고 부상·질병의 가능성과 중대성을 고려해 위험성 수준을 결정하고 감소대책을 수립·실행하는 일련의 과정 (산안법 제36조) |
| HAZOP | Hazard and Operability Study. 공정 변수×가이드워드로 설계 의도로부터의 이탈을 분석하는 공정 위험성평가 기법. 본 시스템에서는 **산업안전보건법상 공정안전자료(PSM)**에 포함되는 공정위험성평가 양식으로 사용 |
| KRAS | 안전보건공단 위험성평가 지원시스템에서 유래한 빈도×강도 기반 평가 방식. 본 시스템에서는 **정기 사업장 위험성평가**에 쓰는 빈도·강도법 시트를 지칭 |
| JSA | Job Safety Analysis. 작업을 단계로 분해하여 단계별 유해·위험요인과 대책을 분석하는 작업 위험성평가 기법. 본 시스템에서는 **신규 작업 발생 시 작업 착수 전** 실시하는 수시평가 양식으로 사용 |
| CHARM | Chemical Hazard Risk Management. 화학물질의 유해성 등급×노출수준 등급으로 위험성을 산정하는 KOSHA 화학물질 평가 기법. 본 시스템에서는 **저장·취급 화학물질**에 대한 물질위험성평가 양식으로 사용 |
| 회사 표준 매트릭스 | 4개 기법에 공통 적용하는 4×4(1~16점) 위험성 결정 매트릭스. 5단계 등급(매우위험 13~16/위험 10~12/다소위험 7~9/위험성존재 5~6/허용가능 1~4)과 등급별 조치기한을 정의 (§5.2.1) |
| 상시평가 | 월(요인 파악·평가)·주(공유·이행점검)·일(작업 전 주지) 주기로 실시하여 수시·정기평가를 갈음하는 제도 (고시 제15조) |
| 수시평가 트리거 | 수시평가 실시 사유 6종(설비 도입·변경, 정비·보수, 작업방법 변경, 재해 발생 등) |
| 감소대책 우선순위 | 제거·대체 > 공학적 대책 > 관리적 대책 > 개인보호구 순의 법정 우선순위 (고시 제12조) |
| 아차사고 | 부상·질병으로 이어지지 않았으나 그럴 가능성이 있었던 사건. 수시평가 사유이자 상시평가 파악 대상 |
| 도급인/수급인 | 자사(도급인)와 협력사(수급인). 수급인은 자기 작업의 평가를 실시하고 도급인은 이를 확인할 의무가 있다 |
| 표준모델 | 작업/공정 유형별로 버전 관리되는 표준 평가 템플릿(요인·대책 세트 포함). 수시평가 이행점검(반기)에서 실사용 평가서를 점검해 개정하는 피드백 루프로 관리된다(§5.3 FR-ASMT-014, §5.6 FR-SCHD-013) |

## 14. 부록 — 실제 업무 흐름 참고 (KDHC 절차서 기준)

> `PRD_추가보완_절차서반영.md`가 정리한 절차서상의 실무 흐름이다. §5의 신규/보강 요구사항이 어떤 흐름의 어느 단계에 대응하는지 확인하는 참고 자료로 둔다.

### 14.1 정기 위험성평가 연간 사이클 (절차서 8장, 별표1.2)

```
[본사] 전사 실시계획 수립·전 사업장 송부 (5월)
  → [지사] 본사 계획을 반영한 자체 실시계획 수립·보고 (6월)  ※ 필요시 지사 계획 先보고 후 본사 반영
  → 사전준비(안전보건정보·기존 평가자료) → 공정분류 → 유해위험요인 파악(순회점검·근로자 면담·재해/아차사고)
  → 위험성 추정·결정 → 감소대책 수립 → [부장] 확인·승인
  → 결과 보고([안전관리관] → [지사장]) → 근로자·수급업체 공유
  → 감소대책 조치 → 조치 후 허용가능 여부 재확인 → [부장] 확인·승인
  → 환류: 조치결과 반기 1회 이상 취합 → [지사장] 적정성 최종 확인
      → 미조치 사항은 차년도 실시계획에 반영
  → (지사→본사) 이행결과 보고 → 전사 실시결과 보고
```

- 역할 분담(절차서 5장): **안전보건관리책임자(지사장)** 총괄·최종 승인·적정성 최종 확인 / **안전관리관** 계획 수립·결과 작성·취합 보고·산업안전보건위원회 보고 / **관리감독자(부장)** 단계별 확인·승인 / **관리감독자(차·팀장)** 평가 실시·수급업체 평가 검토·결과 공유 / **근로자** 사전준비·전 과정 참여·조치결과 작성·기록 보존.
- 계획에 포함할 항목: 평가 일정, 평가 기법, 평가 인원, **평가 전 교육 실시 방안**, 해당연도 본사 주관부서 실시계획.
- 대응 요구사항: FR-SCHD-010~012 (§5.6)

### 14.2 도급 시 수시 위험성평가 사이클 (절차서 9장, 별표1.3)

```
계약 체결 시 [도급인→수급인] 안전보건정보 제공 (전자계약시스템·공문 등)
   : 사업장 위험성평가 결과, 유해위험물질 정보·MSDS, 작업환경측정 결과, 비상경보체계·대피방법
  → 착수(공) 전 [수급인] 도급인 평가 결과를 반영한 위험성평가 실시표 작성·제출
  → [도급인 차·팀장] '수급업체 위험성평가 이행점검 체크리스트'(별표1.6)로 적정성 점검, [부장] 확인
      → 보완 필요 시 수급인에게 수정 요청 또는 작업 전 수시평가에 반영
  → 작업 전 [도급인+수급인 공동] 수시 위험성평가서 확정
  → 안전작업허가서 발행 시 수시 위험성평가서 첨부, 작업현장 비치
  → 작업 시 [작업자·감독자] 작업단계별 '현장확인' 체크 + TBM 공유 여부 확인
      → 현장에서 추가 위험 발견 시 재평가 후 재비치
  → 반기 1회 이상 '수시 위험성평가 이행점검 체크리스트'(별표1.7) 작성
      → 기실시 수시평가서를 점검하여 **표준모델(작업별 표준 평가 템플릿)에 반영·개정**
```

- 대응 요구사항: FR-SHRE-008(§5.8), FR-APRV-009(§5.5), FR-ASMT-J-005/006(§5.4.2), FR-ASMT-014·FR-SCHD-013(§5.3, §5.6)

### 14.3 기법 운용 (절차서 6장)

| 실시 유형 | 절차서 지정 기법 |
|---|---|
| 최초·정기 | (일반) KRAS, (작업) JSA, (물질) CHARM |
| 수시 | (작업) JSA, 체크리스트법, (물질) CHARM |
| 공정 | 정성적(HAZOP, K-PSR), 정량적(CA), 화학물질(CHARM), 작업(JSA) 활용 |

- 정해진 기법 외 타 기법 적용 시 **해당 기법 지침을 작성(안전관리관)·승인(안전보건관리책임자)한 후** 실시해야 한다(§5.2.3 기타 차이 표 참고).
- 체크리스트법·K-PSR·CA는 본 PRD 범위 밖이다(§11).
