위험성평가
통합 관리 시스템

엑셀·수기로 흩어진 위험성평가를 작성 → 결재 → 공유 → 이행 → 기록까지 하나의 시스템으로. 자사와 협력사가 함께 쓰는 법정 의무 대응 플랫폼.

지침서(절차서 R7)가 기준이고, 시스템은 그 기준을 그대로 담는다.
01 · 과제 개요

무엇을, 왜 만드는가

산업안전보건법 제36조는 위험성평가 실시와 3년 기록·보존을 의무화하고, 중대재해처벌법은 그 기록을 핵심 증빙으로 요구합니다. 현재 엑셀·수기 관리로는 복수 사업장·다수 협력사의 평가 이력과 개선조치 이행을 추적하기 어렵습니다. 회사 절차서(KDHC-SHP-04 R7)의 실무 기준을 그대로 담은 통합 시스템을 만듭니다.

평가 작성 4개 기법 평가서 HAZOP·KRAS·JSA·CHARM을 공통 엔진 + 기법별 시트로. 4×4 매트릭스(Ⅴ~Ⅰ), 5점 이상 감소대책 필수 자동 판정.
전자결재 절차서 결재 계층 차·팀장 작성 → 부장 확인·승인 → 안전관리관 보고 → 지사장 최종 승인. 협력사 제출·검토·확인 플로우 포함.
이행·공유 개선조치 + 열람 서명 감소대책이 자동으로 조치 과제가 되고, 증빙 사진·확인 승인으로 닫힙니다. 근로자 모바일 열람·전자서명.
기록·관제 3년 보존 + 대시보드 불변 기록·감사 로그·증빙 패키지 출력. 본사/사업장/협력사별 이행 현황 대시보드.

사용자 (절차서 5장 기준 8개 페르소나)

안전보건관리책임자(지사장) · 본사 위험성평가 주관부서 · 안전관리관 · 관리감독자(부장) · 관리감독자(차·팀장) · 근로자 · 협력사 관리자 · 협력사 근로자 — 결재·확인·보고 권한을 절차서 책임과 권한 그대로 매핑했습니다.

02 · 재점검 반영

2일차에 배운 것으로 다시 본 세 갈래

1일차 기획안에서 비어 있거나 막연했던 데이터·기술·산출물 갈래를 오늘 배운 전처리·하네스·문서 작업으로 채우고 고쳤습니다.

데이터 — 전처리를 반영 1일차: "회사 절차서·서식 확보 필요" (형태 미정, 갈래 비어 있음)
보강: HWPX 절차서·지침서를 zip+XML 파싱으로 텍스트 추출하는 전처리 확립 — 실제로 절차서 R7 전문을 추출해 PRD 정합화에 사용 완료. 추출한 별표 데이터(공정분류 별표1.5, 요인분류 별표2.2, H-code 매핑 별표5.7, 위험성 기준 별표1.1)를 시스템 마스터 데이터로 변환할 계획. 평가서에 근로자 이름·서명이 들어가므로 샘플 데이터는 비식별화해 사용.
기술 — 하네스와 스킬을 반영 1일차: "AI 도구 활용" 수준의 막연한 기술 계획
보강: Claude Code 하네스를 기획·개발 전 과정의 작업 도구로 확정 — PRD v2.0 → Rev1까지 절차서 대조·개정·깃 관리를 실제로 하네스로 수행. 반복 작업은 스킬화 후보로: ① HWPX 전처리 ② PRD-지침서 기준 대조 점검 ③ 기준차이 비교표 생성. 산출물은 깃허브(ksw1727/day10)로 버전 관리.
산출물 — 문서 작업을 반영 1일차: PRD 초안 + HTML 기획안 (단발성 문서)
보강: 반복 생산 가능한 문서 체계로 재정의 — PRD_origin_Rev1(결정 반영·변경부 표시), 기준차이 확인필요사항(4단 비교표), 본 기획안 보강판. 발표 자료는 PPT 프롬프트로 기획안에서 반복 생성. 문서 간 근거 링크(절차서 조항 번호)를 남겨 추적 가능하게 유지.
