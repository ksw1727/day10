# 위험성평가 시스템 — 결정 사항 체크리스트

- 문서 버전: v1.0
- 작성일: 2026-07-08
- 기준 문서: `PRD.md` v1.1, `기획안.html`(3일 MVP)

## 이 문서의 목적

`PRD.md`에는 "설계 단계 결정 사항", "관리자가 수정할 수 있는 설정형", "확정 필요" 등으로 **의도적으로 열어둔 항목**이 여러 곳에 있습니다. 이건 PRD가 미완성이라서가 아니라, 기술 스택·파일 포맷 같은 건 요구사항이 아니라 **제품 책임자(당신)가 내리는 결정**이기 때문입니다.

이 문서는 그 결정 지점들을 PRD 전체에서 모아 문답식으로 정리한 것입니다. 각 질문 아래 `→ 결정:` 줄에 답을 채우면, 그대로 PRD 개정판(v1.2)이나 설계 문서에 반영할 수 있습니다.

**우선순위 표시**: 🔴 지금(3일 MVP 또는 다음 스프린트) 안 정하면 개발이 막히는 항목 / 🟡 Phase 1 이후 정해도 되는 항목 / ⚪ 법무·안전부서 등 외부 확인이 필요한 정책 항목 (PRD §13 오픈 이슈 원문)

## 1. 기술 스택 (PRD §6 각주: "기술 스택은 본 PRD에서 지정하지 않는다")

### Q1-1. 백엔드 언어·프레임워크 🔴
- **맥락**: 3일 MVP는 Node.js + Express로 고정(기획안 §6). PRD 전체 구현 시에도 이걸 계속 쓸지, 아니면 팀 성장에 맞춰 바꿀지는 PRD가 정하지 않았습니다.
- **선택지**: (A) Node.js + Express 계속 사용 (B) 다른 스택으로 전환(NestJS, Spring, Django 등)
- → 결정:

### Q1-2. 데이터베이스 🔴
- **맥락**: MVP는 SQLite(파일 기반). NFR-001은 등록 사용자 3,000명·사업장 20개·협력사 200개·연간 평가서 1만 건·첨부 연 50만 장 규모를 요구합니다. SQLite는 동시 쓰기·다중 서버 확장에 한계가 있습니다.
- **선택지**: (A) SQLite 계속(단일 서버·트래픽이 크지 않다고 판단) (B) PostgreSQL/MySQL 등 서버형 DB로 전환 (C) MVP는 SQLite, Phase 1 정식 구현 시 서버형 DB로 마이그레이션
- → 결정:

### Q1-3. 프런트엔드 방식 🟡
- **맥락**: MVP는 바닐라 JS(기획안 §6 — 팀 학습 수준 때문). 화면 수가 늘어나는 Phase 2~3(HAZOP·CHARM·협력사 포털·대시보드 고도화)에서도 바닐라 JS로 유지 가능한지 판단이 필요합니다.
- **선택지**: (A) 바닐라 JS 유지 (B) 경량 프레임워크 도입(예: Vue, Svelte) (C) 정식 SPA 프레임워크 도입(React 등)
- → 결정:

### Q1-4. 인증 방식 🔴
- **맥락**: PRD §11 범위 제외에 "사내 SSO·인사시스템 연동"이 빠져 있어 **자체 로그인**은 확정입니다. 다만 세션 기반인지 토큰(JWT) 기반인지는 미정입니다(NFR-005: "세션/토큰 만료 정책"이라고만 명시).
- **선택지**: (A) 서버 세션(쿠키) (B) JWT 토큰 (C) 하이브리드(웹은 세션, 모바일 앱 API는 토큰 — 단, 네이티브 앱은 범위 외이므로 우선순위 낮음)
- → 결정:

### Q1-5. 첨부파일 저장소 🔴
- **맥락**: NFR-001의 "첨부(사진 위주) 연 50만 장" 규모를 고려하면 DB에 파일 자체를 저장(BLOB)하는 것과, 파일시스템/오브젝트 스토리지에 두고 DB에는 경로만 저장하는 것 중 선택이 필요합니다.
- **선택지**: (A) 로컬 파일시스템 + DB엔 경로만 저장 (B) S3 호환 오브젝트 스토리지(NCP Object Storage, MinIO 등) + DB엔 경로만 저장 (C) DB에 BLOB으로 직접 저장(소규모까지만 권장)
- → 결정:

### Q1-6. 배포 환경 🟡
- **맥락**: 사내 시스템이므로 온프레미스/사내 서버 배포와 클라우드 배포 중 선택이 필요합니다. 협력사(외부 업체) 접속을 허용하므로 네트워크 접근 정책과도 연결됩니다.
- **선택지**: (A) 사내 서버(온프레미스), VPN 또는 방화벽 예외로 협력사 접근 허용 (B) 클라우드(AWS/NCP 등) 배포, 공인 도메인으로 협력사 직접 접속
- → 결정:

## 2. 출력 파일 포맷

### Q2-1. 평가서 인쇄용 PDF 생성 방식 🔴
- **맥락**: FR-ASMT-009 "평가서 출력: 인쇄용 양식(PDF)과 엑셀 다운로드. 출력물에는 결재 이력·참여자·서명 현황이 포함된다." 포맷은 PDF로 정해져 있지만 **생성 방식**은 미정입니다.
- **선택지**: (A) 브라우저 인쇄(Ctrl+P)용 전용 CSS 페이지만 제공, PDF 변환은 사용자가 직접(가장 단순, MVP에 적합) (B) 서버에서 HTML→PDF 변환 라이브러리(Puppeteer, wkhtmltopdf 등)로 실제 PDF 파일 생성 (C) PDF 생성 라이브러리(PDFKit 등)로 코드상에서 레이아웃을 직접 그림
- → 결정:

### Q2-2. 엑셀 다운로드 포맷 🟡
- **맥락**: FR-ASMT-009(평가서), FR-DASH-006(통계), FR-REC-004(증빙 패키지)가 모두 "엑셀"을 언급하나 정확한 파일 형식은 미정입니다.
- **선택지**: (A) `.xlsx`(서식·색상 유지 가능, 라이브러리 필요: exceljs 등) (B) `.csv`(구현 간단하나 서식·병합 셀 표현 불가)
- → 결정:

### Q2-3. 증빙 패키지(FR-REC-004) 출력 단위 🟡
- **맥락**: "평가서(결재·참여·서명 포함) + 조치 이행 이력 + 상시평가 사이클 기록을 일괄 PDF/엑셀로 출력"이라고 되어 있는데, 이걸 문서 1개로 합칠지 여러 개로 나눌지 미정입니다.
- **선택지**: (A) 평가서별 개별 PDF + 전체를 묶은 ZIP (B) 모든 내용을 하나의 통합 PDF로 병합 (C) PDF(문서류) + 별도 엑셀(집계표) 조합
- → 결정:

## 3. 데이터베이스/파일 저장 형식 (첨부 파일)

- **맥락**: 요인 행 사진, 개선조치 전/후 증빙사진 등 모바일 카메라 직접 업로드를 지원해야 합니다(연 50만 장 규모).
- **결정할 항목**: 허용 확장자(jpg/png/heic 포함 여부 — 아이폰은 기본 heic), 업로드당 최대 용량, 서버 측 리사이즈·압축 여부(원본 보존 필요 여부), 파일명 규칙(평가서번호-행번호-순번 등)

### Q3-2. MSDS 파일 (CHARM, FR-ASMT-C-001) 🟡
- **맥락**: 화학물질 마스터에 MSDS 파일을 첨부합니다. MSDS는 통상 PDF로 유통됩니다.
- **선택지**: (A) PDF만 허용 (B) PDF + 이미지(스캔본) 허용
- → 결정:

### Q3-3. 도면(P&ID) 파일 (HAZOP, §5.4.3) 🟡
- **맥락**: "참조 도면(P&ID 번호·파일 첨부)". CAD 원본 파일까지 받을지, 열람용 이미지/PDF만 받을지 결정이 필요합니다.
- **선택지**: (A) PDF/이미지만 허용(열람 목적) (B) CAD 원본(dwg 등) 첨부도 허용

### Q3-4. 실시규정 문서 (FR-CFG-001) 🟡
- **맥락**: "위험성평가 실시규정 문서를 등록·게시(파일 첨부 + 시행일 + 개정 이력)". 형식 제한이 없습니다.
- **선택지**: (A) PDF만 허용(사내 공식 문서 관례) (B) Word/한글(hwp) 포함 다양한 형식 허용
- → 결정:

### Q3-5. 근로자 대량 등록 업로드 (FR-ORG-009) 🟡
- **맥락**: "근로자 대량 등록(엑셀 업로드)". 업로드 템플릿의 정확한 컬럼 구성이 필요합니다.
- **결정할 항목**: 템플릿 컬럼 순서·필수값, 중복(이미 등록된 근로자) 처리 방식(건너뛰기/덮어쓰기/오류), 업로드 실패 행 처리(부분 성공 허용 여부)
- → 결정:

### Q3-6. 전자서명 데이터 형식 (FR-SHRE-003) 🔴
- **맥락**: "전자서명(터치 서명 또는 확인 버튼+본인인증)". 터치 서명을 채택할 경우 손글씨 이미지를 어떤 형식으로 저장할지 결정이 필요합니다. (법적 증빙력 수준 자체는 §13 이슈2 — 아래 5장 참조)
- **선택지**: (A) 터치 서명을 PNG/SVG 이미지로 저장 (B) "확인" 버튼 클릭 + 사용자 인증 정보만 로그로 저장(이미지 없음)
- → 결정:
