1전체 아키텍처 개요

3일 MVP에서 지금 구현하는 스택과, PRD 전체 로드맵(§10)에서 필요해지는 확장 지점을 함께 표시합니다. 확장 박스는 방향성이며 구체 기술은 PRD상 설계 단계 결정사항입니다(§6 각주: "기술 스택은 본 PRD에서 지정하지 않는다").

Phase 1 — 코어KRAS·JSA 작성 + 결재 + 조치 이행 + 공유·서명 + 기록3일 MVP는 이 중 KRAS 1개 기법 + 승인/반려 1단계만 구현. 조치이행·공유서명은 제외

3일 MVP는 위 아키텍처의 "지금 구현" 칸만으로 완결되는 가장 얇은 조각입니다. 기법이 늘고(JSA→CHARM→HAZOP), 결재선·이행관리·공유서명·일정관리가 붙을 때마다 서버 계층의 "확장 방향" 박스가 하나씩 실선으로 바뀐다고 생각하면 됩니다.

위험성 점수등급조치 요구사항

function calcRisk(frequency, severity) {
  const score = frequency * severity;   // 1~16
  const needsAction = score >= 5;       // 5점 이상 = 감소대책 필수
  return { score, needsAction };
}

fetch + async/await

필수

왜화면이 서버 API(기획안 §6 API 목록)와 통신하는 유일한 방법입니다.

어디서public/js/api.js.

async function createAssessment(data) {
  const res = await fetch('/api/assessments', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data),
  });
  if (!res.ok) throw new Error('저장 실패');
  return res.json();
}

SQL 기초 (CREATE/SELECT/INSERT/UPDATE, 외래키)

필수

왜평가서 1건에 요인 행이 여러 개 딸리는 1:N 구조를 표현해야 합니다 (PRD §5.3).

어디서server/db.js.

CREATE TABLE assessments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT, department TEXT, evaluator TEXT,
  status TEXT DEFAULT 'draft', reject_reason TEXT,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
CREATE TABLE assessment_items (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  assessment_id INTEGER REFERENCES assessments(id),
  hazard TEXT, current_measure TEXT,
  frequency INTEGER, severity INTEGER,
  reduction_measure TEXT
);

시간이 남으면 배워서 적용해볼 것들.

트랜잭션

선택

왜평가서 헤더 저장과 요인 행 여러 개 저장을 한 묶음으로 처리해, 중간에 실패해도 데이터가 반쪽만 남지 않게 합니다.

어디서server/routes/assessments.js(POST 처리).

const insertMany = db.transaction((assessmentId, items) => {
  const stmt = db.prepare(
    'INSERT INTO assessment_items (assessment_id, hazard, frequency, severity) VALUES (?, ?, ?, ?)'
  );
  for (const item of items) {
    stmt.run(assessmentId, item.hazard, item.frequency, item.severity);
  }
});

서버 측 입력값 검증

선택

왜프론트의 "감소대책 필수" 검증을 우회해도(개발자도구 등) 서버가 5점 이상 항목을 감소대책 없이 저장하지 않도록 막아야 합니다 (PRD FR-ASMT-003).

어디서server/routes/assessments.js.

for (const item of items) {
  if (item.frequency * item.severity >= 5 && !item.reduction_measure) {
    return res.status(400).json({ error: '위험성 5점 이상 항목은 감소대책이 필요합니다.' });
  }
}
